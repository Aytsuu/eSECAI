# Library Rules

## Structure
- Use this directory for utility functions, configuration files, or external library integrations that need to be centralized.
- Commonly used for `shadcn/ui` utility functions (`utils.ts`).

## Content
- Keep utility functions related to the project's core functionality here (e.g., class name management with `clsx` and `tailwind-merge`).
- Avoid putting business logic or component-specific code here.

## Naming
- Use clear and descriptive filenames (e.g., `utils.ts`).

## Exports
- Use named exports for utilities.
